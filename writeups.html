Category: SOC Analysis Date: December 2024 Author: Shubhdeep Singh

üìã Summary

Real-world incident where a brute-force attack was detected and mitigated using Splunk SIEM. The attack targeted SSH services on a production Linux server, with over 5,000 failed login attempts detected within a 2-hour window.

‚ö†Ô∏è Key Finding: Attacker attempted to compromise SSH access using credential stuffing from IP address 185.220.101.45 (TOR exit node).

‚è±Ô∏è Attack Timeline

  • 14:23 UTC - First failed login attempt detected
  • 14:25 UTC - Automated Splunk alert triggered
  • 14:30 UTC - SOC analyst begins investigation
  • 15:45 UTC - Source IP blocked at firewall level
  • 16:20 UTC - Attack ceases, total: 5,247 attempts

üîç Detection Method

The attack was detected using a custom Splunk query monitoring authentication logs:

index=linux sourcetype=linux_secure "Failed password"
| stats count by src_ip, user
| where count > 10
| sort -count

üìä Analysis Results

  • Attack Vector: SSH (Port 22)
  • Source IP: 185.220.101.45 (TOR exit node, Germany)
  • Target Usernames: 47 unique usernames attempted
  • Attack Rate: Average 45 attempts per minute

üí° Recommendations

  • Implement rate limiting on SSH service (fail2ban)
  • Enable MFA for all SSH access
  • Disable root login via SSH
  • Change SSH default port from 22 to non-standard port

‚úÖ Conclusion

The brute-force attack was successfully detected and mitigated with no compromise of systems or data.
